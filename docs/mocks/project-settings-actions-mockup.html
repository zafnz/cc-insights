1. Project Settings — User Actions (Commands + Macros)

The "User Actions" category now supports two action types: Command (run a shell script) and Start Chat (create a new chat with a specific agent, model, and instruction). Each action appears as a card with a type badge. The "Add Action" button opens a dropdown to choose the type.

Actions appear as buttons in the Actions panel. Command actions run shell scripts in the worktree directory. Start Chat macros create a new chat with the specified agent and send the instruction automatically.

2. Actions Panel — Button Rendering

Command actions show a play icon. Start Chat macros show a chat bubble icon with a subtle purple accent border. Tooltips show the command or instruction preview on hover.

3. JSON Config Format

The .ccinsights/config.json format. String values are backward-compatible command actions. Object values with a type field define macros. 

// .ccinsights/config.json
{
  "user-actions": {
    "Test": "./test.sh",            // string = command (backward compat)
    "Run": "./run.sh",
    "Codex Review": {             // object = macro
      "type": "start-chat",
      "agent-id": "codex-default",
      "model": "o3",
      "instruction": "Perform a code review..."
    },
    "Suggest Refactors": {
      "type": "start-chat",
      "agent-id": "claude-default",
      "model": "opus",
      "instruction": "Review the codebase..."
    }
  }
}
4. Empty Macro — Just Added

When a user clicks "Add Action > Start Chat", a new empty macro card appears with the agent dropdown defaulting to the first available agent and the instruction field focused.
