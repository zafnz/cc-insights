## Contributing to CC Insights

All contributions are welcome, but please create an issue ticket _before_ you begin your work to get feedback as to whether or how the feature should be implemented.

## Use of AI
**Use it. Duh.** **Seriously**, unless you're solving some very odd bug by hand, you should _only_ be contributing code that has been written by AI. 

## Testing
This is the only reason why this product isn't finished yet, testing the features requires human activity, trying things out, seeing how they work, submitted feedback to 
the agent, continuing the cycle. If you can optimize this part of the loop then you will make a lot of money. 

At the moment with the orchestration tool my entire time is spent testing the dozen or so features and bug fixes the AI just implemented. Unit tests and integration tests
can only go so far. Especially since what you think you want isn't always what you actually want, and you don't discover that till you get what you don't want.
